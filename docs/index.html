Project Overview

Vision

What’s for Dinner is a web application that quickly finds recipes well-suited to ingredients users have on hand. It combines user-provided input (ingredients and quantities) with required “kitchen staples” (salt, pepper, oil, etc.) to suggest possible recipes that can be prepared without a trip to the grocery store.

Motivation

Many students struggle to coordinate group work across different schedules, time zones, and course loads, often relying on fragmented tools like shared documents, group chats, and calendars. StudySync aims to simplify this process by providing a single, purpose-built platform that makes planning, accountability, and communication more transparent for student teams.

Risk & Mitigation

Risks to completion

  • Scheduling and Coordination Challenges

    Team members are in different time zones, which may limit synchronous development time. Also Team members have different work schedules making it hard to coordinate.

  • Learning Curve with Full-Stack Integration

    Some team members have limited experience connecting frontend interfaces with backend APIs and databases.

Mitigation Strategies

  • Availability Alignment

    The team will rely on asynchronous communication tools and keep synchronous meetings focused and time-boxed in order to meet everyone's schedule.

  • Minimum Viable Product First

    The initial goal will be a basic, functional product that allows for basic recipes to be output.

Development Plan

Development Methodology

The team will follow an Agile, sprint-based approach with weekly planning meetings, short development iterations, and continuous integration. Tasks will be broken into small, testable components, allowing the project to adapt based on progress and challenges.

Developmental Milestones

  1. Define functional and pre-development requirements
    1. Define user stories
    2. Find targets for web scraping
    3. Identify tools/APIs
      1. What language will back end logic be in?
      2. How will we host the SQL server?
      3. How will we send the HTTP request and what will be at POST?
    4. Create project website
  2. Fulfill pre-development requirements and define database schema
    1. Implement web scraper
    2. Plan schema
      1. Data Warehouse or Relational?
      2. Primary and foreign keys
  3. Build and initialize SQL database
    1. Implement table schema and begin populating tables
    2. Test data retrieval directly against the database.
  4. Develop backend API and finalize front end
    1. Create and test scripts that can run the defined queries.
    2. Finalize look and interactables on project front end.
  5. Connect front and back end with an HTTP endpoint
    1. Test vs the user stories

Project Tracking

Trello will be the primary tracking tool for this project. Tasks will be assigned to team members each week with due dates, and lists will be an easy point-of-reference for weekly meetings. This will streamline communication about weekly progress and provide clarity on each team member’s responsibilities for that week.

Planned Tech Stack

Front End

  • HTML & CSS
  • JavaScript with React

Back End

  • Python with Flask for RESTful API development
  • Python and Batch for web scraping (if implemented)

Database

  • SQL Lite - runs as part of the application for these testing phases.
